# ToolCase v5.5 ‚Äî Kwaliteits-assistent Roadmap

> **Goal:** Transformeer ToolCase van "57 losse scanners" naar een betrouwbare kwaliteits-assistent met duidelijke prioriteit, reproduceerbaarheid en veilige workflows.

**Architecture:** Nieuwe modules voor output-standaarden (SARIF), scan-betrouwbaarheid (reliability report, baseline mode), pre-flight checks (release readiness, sandbox scanner). Bestaande scanners worden gewrapt met error accounting en risk scoring.

**Tech Stack:** Python 3.11+, stdlib-only, SARIF v2.1.0 JSON schema, GitHub Code Scanning integratie.

### Task 1.3: SARIF Exporter (`sarif_exporter.py`)

**Objective:** Converteer ToolCase security findings naar SARIF v2.1.0 formaat voor GitHub Code Scanning integratie.

**Files:**
- Create: `D:/ToolCase/sarif_exporter.py`
- Create: `D:/ToolCase/schemas/sarif_v2.1.0.schema.json` (reference)
- Modify: `D:/ToolCase/tools_config.json` (tool #60)

**SARIF mapping:**
| ToolCase Risk | SARIF Level | SARIF Rank |
|---|---|---|
| HIGH | error | 9.0 |
| MEDIUM | warning | 6.0 |
| LOW | note | 3.0 |

**Usage:**
```bash
python security_scan.py . --json | python sarif_exporter.py --output toolcase.sarif
# Of direct:
python sarif_exporter.py --scan security --path . --output toolcase.sarif
```

**SARIF properties per finding:**
- `ruleId`: pattern name (e.g. `api_key`, `eval_exec`)
- `message.text`: fix suggestion
- `locations[].physicalLocation.artifactLocation.uri`: relative file path
- `locations[].physicalLocation.region.startLine`: line number
- `partialFingerprints.primary`: SHA256 of file+line+pattern for dedup

**CI integratie:**
```yaml
- name: Security scan
  run: python sarif_exporter.py --scan security --path . --output toolcase.sarif
- name: Upload SARIF
  uses: github/codeql-action/upload-sarif@v3
  with:
    sarif_file: toolcase.sarif
```

### Task 2.1: Package Sandbox Scanner (`skill_sandbox.py`)

**Objective:** Pre-installatie scanner die skill packages analyseert op supply-chain risico's.

**Files:**
- Create: `D:/ToolCase/skill_sandbox.py`
- Modify: `D:/ToolCase/skill_installer.py` (integrate sandbox check pre-install)

**Checks:**
1. Symlinks ‚Äî detect, resolve, containment check
2. Executable files ‚Äî .sh, .py, .exe, .bat met +x
3. Network calls ‚Äî `requests`, `urllib`, `socket`, `http.client` imports
4. Subprocess ‚Äî `subprocess`, `os.system`, `os.popen` usage
5. Suspicious imports ‚Äî `ctypes`, `pickle`, `eval`, `exec`, `compile`
6. Filesystem writes ‚Äî `open(..., 'w')`, `shutil.rmtree` buiten target
7. Metadata trust ‚Äî source URL, author, signature verificatie

**Output:** Risk score per category, GO/NO-GO voor installatie, `--json` mode.

### Task 2.2: Baseline Mode (`baseline.py`)

**Objective:** Eerste scan wordt baseline ‚Äî daarna alleen nieuwe/delta findings.

**Files:**
- Create: `D:/ToolCase/baseline.py`

**Mechanism:**
1. Eerste run: `python baseline.py init --scan security --path .`
   - Draait `security_scan.py` en slaat SHA256 fingerprint per finding op in `.toolcase/baseline.json`
2. Volgende runs: `python baseline.py check --scan security --path .`
   - Draait scan, vergelijkt met baseline, toont alleen **nieuwe** findings
3. `python baseline.py update` ‚Äî accepteer huidige staat als nieuwe baseline
4. `python baseline.py diff` ‚Äî toon delta (nieuw, verdwenen, ongewijzigd)

**Fingerprint:** `SHA256(filepath + line + pattern + risk)`
